Colector centralizado de métricas y logs que actúa como intermediario entre tus aplicaciones y servicios de monitoreo.

Procesamiento asíncrono
Conexión lazy a MongoDB
Envío a DataDog
Respuestas inmediatas (HTTP 202)
📖 Swagger UI - Documentación Interactiva /apidocs/ →
💧 Monitor de Estanque - Posada en el Bosque /monitor →

API Endpoints

GET / Documentación y estado de la API
POST /metrics Enviar métricas 
{
  "series": [
    {
      "metric": "mi.metrica.contador",
      "points": [[1730745173, 2]],
      "tags": ["host:localhost", "environment:develop"],
      "host": "localhost"
    }
  ]
}
Response: 202 Accepted — {"message": "Metric recibido y en proceso"}
POST /log Enviar un log individual 
{
  "message": "Un evento ocurrió",
  "level": "info",
  "service": "mi-servicio",
  "ddsource": "python",
  "hostname": "localhost",
  "tags": ["env:production"],
  "date": "2024-11-04T15:35:56"
}
Response: 202 Accepted — {"message": "Log recibido y en proceso"}
POST /logs Enviar múltiples logs 
{
  "logs": [
    {
      "message": "Primer log",
      "level": "info",
      "service": "mi-servicio",
      "ddsource": "python",
      "hostname": "localhost",
      "tags": ["env:test"]
    },
    {
      "message": "Segundo log",
      "level": "error",
      "service": "mi-servicio",
      "ddsource": "python",
      "hostname": "localhost",
      "tags": ["env:production"]
    }
  ]
}
Response: 202 Accepted — {"message": "Todos los logs han sido recibidos y estan en proceso"}

Configuración

Variables de entorno necesarias en el archivo .env:

Variable Descripción Default
MONGODB_URI URI de conexión a MongoDB -
ENABLE_MONGO_DB Habilita almacenamiento en MongoDB false
DATADOG_API_KEY API Key de DataDog -
ENABLE_SEND_LOG_TO_DATADOG Habilita envío de logs a DataDog false

Ejemplos con cURL

curl -X POST http://localhost:5000/metrics \
  -H "Content-Type: application/json" \
  -d '{
    "series": [{
      "metric": "test.counter",
      "points": [[1730745173, 1]],
      "tags": ["env:dev"],
      "host": "localhost"
    }]
  }'
curl -X POST http://localhost:5000/log \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Test log message",
    "level": "info",
    "service": "my-service",
    "ddsource": "python",
    "hostname": "localhost",
    "tags": ["env:dev"]
  }'
